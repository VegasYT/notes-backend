## Что делал с помощью AI и почему

### Тестовая среда (tests/)

Тестовую среду настраивал раньше, но не часто - боялся ошибиться в деталях. Конкретно не был уверен как правильно подменить PostgreSQL на SQLite, отключить lifespan и как работает httpx с ASGI в связке с pytest-asyncio.

Что понял в итоге:
- dependency_overrides[get_db_session] = override_db - FastAPI подменяет зависимость, не трогая бизнес-логику. Достаточно передать функцию с тем же интерфейсом
- ASGITransport(app=app) - httpx отправляет запросы напрямую в ASGI-приложение без реального HTTP-сервера и открытого порта
- noop_lifespan - заменяет настоящий lifespan (который пытается подключиться к Redis/Kafka) на пустой контекстный менеджер, тесты работают без внешних сервисов
- FakeSessionStore с тем же интерфейсом что SessionStore - Python не требует наследования, достаточно совпадения методов
- rollback() после каждого теста вместо пересоздания БД - быстрее и надёжнее


### MongoDB и Kafka (app/db/mongo_client.py, app/kafka/)

Мало работал с этими технологиями, боялся сделать что-то неправильно - особенно в части async и управления соединениями.

Что понял:
- Motor - async-обёртка над pymongo, нужна потому что стандартный pymongo синхронный и блокировал бы event loop
- Глобальный клиент инициализируется один раз в lifespan - не создаётся на каждый запрос
- asyncio.create_task(_consume_loop()) - запускает consumer параллельно с обработкой запросов в том же event loop
- task.cancel() + await task с перехватом CancelledError - корректный способ остановить долгоживущую asyncio-задачу при завершении приложения
- group_id - Kafka хранит offset (позицию чтения) отдельно для каждой группы, auto_offset_reset="earliest" означает читать с начала топика при первом запуске
- Retry-логика при старте нужна потому что Kafka поднимается дольше чем приложение


### Админ-панель (admin/)

Мало работал с готовыми admin-панелями, не знал как устроен sqladmin и как правильно реализовать аутентификацию через AuthenticationBackend.

Что понял:
- AuthenticationBackend - sqladmin вызывает authenticate() перед каждым запросом к /admin/*. Если возвращает True - пропускает
- authenticate() читает session_id из cookie, проверяет Redis, загружает пользователя и проверяет флаг is_admin - та же логика что в обычных зависимостях
- BaseView + @expose - кастомная страница без привязки к модели, можно делать любые запросы (в нашем случае к MongoDB)
- templates_dir в sqladmin - jinja ищет шаблоны в этой папке, {% extends "layout.html" %} наследует базовый layout sqladmin через PackageLoader
- add_base_view регистрирует кастомную страницу в боковом меню
